There is a layout of the database tables in the sqlTable.sql file

-POST/login

-body {"username": "name of user", "password": "enter password"}

will allow a existing user to login


-POST/createUser

-body {"username": "name of user", "password": "enter password", "confirmPassword": "enter same password"}

will add a new user to our users table in the database

this can only be done by users with admin role


-POST/recipe/add

-body {"recipe_name": "name of recipe", "image": "URL of image to be added", "servings": "number of servings"}

will add a new recipe to our recipe table in the database


-POST/ingredient/add

-body {"ingredient": "name of ingredient", "prep_time": "time amount", "instruction": "say what to do"}

will add a new ingredient to our ingredients table in the database


GET/recipe/list

We are just requesting the entire recipe table, there is no body


GET/ingredient/list

We are just requesting the entire ingredients table, there is no body


DELETE/recipe/:id

enter the id of recipe to be delete in the place of :id

the recipe with that id will be deleted from the database


DELETE/ingredient/:id

enter the id of ingredient to be deleted in the plae of :id

the ingredient with that id will be deleted from the database


PUT/recipe/change

-body {"recipe": "name of recipe you want to change servings on", "servings": "new number of servings"}

will select recipe and then change servings on that selected recipe


PUT/ingredient/change

-body {"ingredient": "name of ingredient you want to change instruction on", "instruction": "new instructions"}

will select ingredient and then change instruction on that selected ingredient


POST/recipe_ingredient/add

-body {recipe_id: "id of recipe", ingredient_id: "id of ingredient"}

will add a new ingredient to our recipe_ingredient table in the database


GET/recipe_ingredient/list

there is no body, we are just requesting the entire recipe_ingredient table

will display the recipe_ingredient table
